아래와 같이 AWS spark로 DB연결후 테이블 조회 10건만 하는 파이썬 소스 생성
beeline "jdpc:hive로시작하는연결정보" -n user -p password

아래는 beeline 대신 **PySpark (AWS Spark 환경)**에서 Hive/JDBC로 DB에 연결해서 테이블 10건만 조회하는 Python 예제입니다.
(beeline 명령을 PySpark 코드로 바꾼 형태)

✅ 1. PySpark + Hive (EMR / Glue / Hive 메타스토어 사용 시)
from pyspark.sql import SparkSession

# SparkSession 생성 (Hive 지원)
spark = SparkSession.builder \
    .appName("HiveTableRead") \
    .enableHiveSupport() \
    .getOrCreate()

# 테이블 조회 (예: db_name.table_name)
df = spark.sql("SELECT * FROM db_name.table_name LIMIT 10")

# 결과 출력
df.show()

spark.stop()

👉 전제

EMR / Glue Data Catalog / Hive Metastore 연결되어 있어야 함
beeline 없이 바로 SQL 사용 가능
✅ 2. JDBC 방식 (beeline 연결 문자열 기반)

beeline "jdbc:hive2://..." 이걸 그대로 PySpark에서 사용

from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("JDBC_Hive_Read") \
    .getOrCreate()

jdbc_url = "jdbc:hive2://host:10000/default"

properties = {
    "user": "user",
    "password": "password",
    "driver": "org.apache.hive.jdbc.HiveDriver"
}

# 방법1: 테이블 직접 조회
df = spark.read.jdbc(
    url=jdbc_url,
    table="db_name.table_name",
    properties=properties
).limit(10)

df.show()
✅ 3. 쿼리 직접 실행 방식 (추천 ⭐)
query = "(SELECT * FROM db_name.table_name LIMIT 10) tmp"

df = spark.read.jdbc(
    url=jdbc_url,
    table=query,
    properties=properties
)

df.show()

"from pyspark.sql import SparkSession"이것 없이 접속하는 방법은

결론부터 말하면 👉 PySpark에서 from pyspark.sql import SparkSession 없이 Spark를 직접 사용하는 방법은 거의 없습니다.
왜냐하면 Spark 2.x 이후부터는 모든 진입점(entry point)이 SparkSession으로 통합됐기 때문입니다.

다만, 목적에 따라 아래 대체 방법 3가지는 가능합니다.

✅ 1. Spark 없이 → pure Python (JDBC 직접 연결)

👉 Spark 안 쓰고 DB만 조회

from pyhive import hive

conn = hive.Connection(
    host="host",
    port=10000,
    username="user",
    password="password",
    database="default"
)

cursor = conn.cursor()
cursor.execute("SELECT * FROM db_name.table_name LIMIT 10")

for row in cursor.fetchall():
    print(row)

👉 특징

beeline과 가장 유사

✅ 2. SparkSession 없이 → SparkContext + SQLContext (구버전 방식)

👉 가능은 하지만 비추천 (레거시 방식)

from pyspark import SparkContext
from pyspark.sql import SQLContext

sc = SparkContext()
sqlContext = SQLContext(sc)

df = sqlContext.read \
    .format("jdbc") \
    .option("url", "jdbc:hive2://host:10000/default") \
    .option("dbtable", "db_name.table_name") \
    .option("user", "user") \
    .option("password", "password") \
    .option("driver", "org.apache.hive.jdbc.HiveDriver") \
    .load()

df.show(10)

👉 특징

SparkSession 없이 가능
하지만 현재는 거의 안 씀
✅ 3. 아예 Python 대신 beeline 그대로 실행 (subprocess)

👉 Python에서 beeline 호출

import subprocess

cmd = [
    "beeline",
    "-u", "jdbc:hive2://host:10000/default",
    "-n", "user",
    "-p", "password",
    "-e", "SELECT * FROM db_name.table_name LIMIT 10"
]

result = subprocess.run(cmd, capture_output=True, text=True)
print(result.stdout)

import 안하고 연결하는 방법은 없는지

짧게 핵심만 말하면 👉
Python에서 import 없이 DB 연결하는 건 “불가능”에 가깝습니다.

이유는 간단합니다:

Python은 외부 기능(네트워크, DB, Spark 등)을 쓰려면 반드시 모듈을 import 해야 합니다.
beeline, JDBC, Spark 전부 다 라이브러리 기반이라서 import 없이 직접 호출할 수 없습니다.
🔥 그래도 가능한 “우회 방법” 2가지
✅ 1. OS 명령으로 직접 실행 (import 없이)

👉 Python 기능 안 쓰고 그냥 쉘 명령 실행

import os

os.system('beeline -u "jdbc:hive2://host:10000/default" -n user -p password -e "SELECT * FROM db_name.table_name LIMIT 10"')

👉 완전 import 없이 하고 싶다면
→ 사실상 Python이 아니라 쉘에서 직접 실행해야 합니다

beeline -u "jdbc:hive2://host:10000/default" -n user -p password \
-e "SELECT * FROM db_name.table_name LIMIT 10"
✅ 2. PySpark 실행 환경 자체를 이용 (코드 최소화)

👉 spark-shell / pyspark 실행 상태에서는 일부 import 생략 가능

df = spark.sql("SELECT * FROM db_name.table_name LIMIT 10")
df.show()

이미 spark 객체가 자동 생성되어 있음
하지만 내부적으로는 이미 import된 상태
❗ 왜 완전 불가능하냐면

Python 구조상:

아래 소스에서 파일호출하는 방식으로 쿼리부분변경하면
import os

os.system('beeline -u "jdbc:hive2://host:10000/default" -n user -p password -e "SELECT * FROM db_name.table_name LIMIT 10"')

좋은 방식이에요 👍
beeline에서도 SQL 파일을 직접 실행하는 옵션이 있어서, 문자열 대신 파일 호출로 바꾸는 게 훨씬 깔끔합니다.

✅ 방법 1. -f 옵션 (가장 추천 ⭐)
1) SQL 파일 생성

예: query.sql

SELECT *
FROM db_name.table_name
LIMIT 10;
2) Python 코드 수정
import os

os.system('beeline -u "jdbc:hive2://host:10000/default" -n user -p password -f query.sql')

👉 핵심

-e → 문자열 실행
-f → 파일 실행
✅ 방법 2. 변수 치환까지 같이 쓰기 (실무형 ⭐⭐⭐)
1) SQL 파일 (템플릿)
SELECT *
FROM db_name.table_name
WHERE dt = '${hivevar:dt}'
LIMIT 10;
2) Python 코드
import os

cmd = '''
beeline -u "jdbc:hive2://host:10000/default" \
-n user -p password \
--hivevar dt=20260330 \
-f query.sql
'''

os.system(cmd)

👉 장점

날짜, 파라미터 동적 처리 가능
배치 작업에 매우 적합
✅ 방법 3. 여러 SQL 파일 실행
import os

sql_files = ["query1.sql", "query2.sql"]

for file in sql_files:
    os.system(f'beeline -u "jdbc:hive2://host:10000/default" -n user -p password -f {file}')
⚠️ 주의사항
1. 경로 문제
-f /full/path/query.sql

👉 상대경로 에러 자주 발생

print(result.stdout)
print(result.stderr)
